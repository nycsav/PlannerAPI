## 📝 Documentation Maintenance Guidelines

### When to Update This File
1. **New documentation created** → Add to "Active Documentation" section with date
2. **Major documentation update** → Update "Last Updated" date in relevant section
3. **Documentation archived** → Move to "Archive" section with archive date
4. **Weekly review** → Verify all dates are current

### Version Control Best Practices
1. **Always include date** in new documentation headers
2. **Format**: `Last Updated: Month DD, YYYY` or `Date: YYYY-MM-DD`
3. **Commit messages** should reference documentation updates
4. **Archive old versions** when superseded by newer docs

### Priority Levels
- 📌 **Critical**: Must read for all development work (CLAUDE.md)
- 💰 **Cost-sensitive**: Related to API usage and billing
- 🎨 **Design**: Visual and UX guidelines
- 🔧 **Technical**: Implementation details
- 📖 **Reference**: Historical or one-time setup
